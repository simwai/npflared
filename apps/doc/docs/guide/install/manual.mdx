# Manual Transcendence: The Traditional Path

### 1. Cloning the Collective Code
To begin the manual installation, clone the project and enter the source directory:

```bash
git clone https://github.com/Thomascogez/babadeluxe-registry.git
cd babadeluxe-registry
```

Alternatively, you may use `degit` to retrieve only the necessary components:
```shell
npx degit Thomascogez/babadeluxe-registry/apps/api
```

## 2. Installing Dependencies
After cloning the source, install the project's dependencies using the `pnpm` manager:
<PackageManagerTabs command="install" />

:::info
This project utilizes `pnpm` as its primary dependency manager.
:::

## 3. Configuring the Nexus
To configure your instance, you must edit the `wrangler.toml` file.

### Configuring R2 Storage
Navigate to your [Cloudflare Dashboard](https://dash.cloudflare.com/) and create a new **R2** bucket.
Inscribe the name of your bucket within your `wrangler.toml` file:

```toml title="wrangler.toml"
[[r2_buckets]]
binding = "BUCKET"
bucket_name = "YOUR_R2_BUCKET_NAME"
```

### Configuring the D1 Database
Create a new **D1** database through the Cloudflare Dashboard.
Copy the ID and name of your database and inscribe them within `wrangler.toml`:

```toml title="wrangler.toml"
[[d1_databases]]
binding = "DB"
database_name = "YOUR_D1_DATABASE_NAME"
database_id = "YOUR_D1_DATABASE_ID"
```

## 4. Deploying the Babadeluxe Worker
Once your configuration is complete, deploy the worker to the Cloudflare network:
<PackageManagerTabs command="run deploy" />

## 5. Running Initial Migrations
After deployment, execute the initial database migrations to prepare the environment:
<PackageManagerTabs command="run migrate:prod" />

## 6. Creating the Admin Token
To manage your domain and distribute tokens, you must create an initial administrative token with elevated privileges.

Run the following command, replacing `ADMIN_TOKEN` with your chosen secret:

```bash
npx wrangler d1 execute DB --remote --yes --command "INSERT INTO token (token, name, scopes, created_at, updated_at) VALUES ('ADMIN_TOKEN', 'admin-token', '[{\"type\": \"token:read+write\", \"values\": [\"*\"]}, {\"type\": \"user:read+write\", \"values\": [\"*\"]}, {\"type\": \"package:read+write\", \"values\": [\"*\"]}]', 1734173399, 1734173399)"
```
