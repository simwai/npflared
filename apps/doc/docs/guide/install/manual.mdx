# Manual Substrate Configuration

For those who wish to manually manifest their BabaDeluxe-registry node, bypassing the automated CLI genesis, follow these steps to establish your registry's foundations.

### 1. Extracting the Manifest
Retrieve the core API substrate from the BabaDeluxe Registry repository.

```shell
npx degit Thomascogez/npflared/apps/api
```

:::info
Using `degit` allows you to isolate the necessary API artifacts without the entire monorepo overhead.
:::

## 2. Ingesting Dependencies
Initialize the necessary node modules to support your registry's logic.

<PackageManagerTabs command="install" />

:::info
BabaDeluxe Registry is architected using `pnpm` as its primary dependency substrate.
:::

## 3. Configuring the Infrastructure
You must now align your `wrangler.toml` with your Cloudflare environment.

### Manifesting the R2 Bucket
Access your [Cloudflare Dashboard](https://dash.cloudflare.com/) and manifest a new **R2** bucket. Update your configuration to link this storage layer.

```toml title="wrangler.toml"
[[r2_buckets]]
binding = "BUCKET"
bucket_name = "BUCKET_NAME" # Replace BUCKET_NAME with the name of your bucket
```

### Manifesting the D1 Database
Create a new **D1** database in your dashboard and integrate its identity into your configuration.

```toml title="wrangler.toml"
[[d1_databases]]
binding = "DB"
database_name = "D1_DATABASE_NAME" # Replace D1_DATABASE_NAME with your database name
database_id = "D1_DATABASE_ID" # Replace D1_DATABASE_ID with your database ID
```

## 4. Deploying the Worker Node
Once the configuration is synchronized, deploy your registry brain to the Cloudflare edge.

<PackageManagerTabs command="run deploy" />

## 5. Synchronizing the Database Schema
Execute the initial migrations to establish the memory structure within your D1 database.

<PackageManagerTabs command="run migrate:prod" />

## 6. Manifesting the Admin Signal (Token)
To manage your registry, you must first create an initial token with elevated permissions. Replace `ADMIN_TOKEN` with a secure, random string.

```bash
npx wrangler d1 execute DB --remote --yes --command "INSERT INTO token (token, name, scopes, created_at, updated_at) VALUES ('ADMIN_TOKEN', 'admin-token', '[{\"type\": \"token:read+write\", \"values\": [\"*\"]}, {\"type\": \"user:read+write\", \"values\": [\"*\"]}, {\"type\": \"package:read+write\", \"values\": [\"*\"]}]', $(date +%s), $(date +%s))"
```

Your manual node manifestation is complete 🎉
