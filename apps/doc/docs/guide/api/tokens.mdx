# Token Management

Tokens are the cryptographic trust-signals that allow you to interact with the BabaDeluxe Registry. Managing these is key to maintaining a secure and functional node for our shared codebases.

## Generating a New Signal (Token Create)

:::info
In order to generate a new token, you will need to be authenticated and having at least the `token:write` permission with value `*`.
:::

You will need to send a `POST` request to the `/-/npm/v1/tokens` endpoint with the following body:

```js
{
	"name": "My token",
	"scopes": [
		{
			"type": "package:read",
			"values": ["*"]
		},
		{
			"type": "package:write",
			"values": ["@my-scope/my-package", "my-cool-package"]
		}
	]
}
```

You can send has many scopes as you want.

| Scope type | Description |
| --- | --- |
| `package:read` | Read access to a package |
| `package:write` | Write access to a package |
| `package:read+write` | Read and write access to a package |
| `token:read` | Read access to a token |
| `token:write` | Write access to a token |
| `token:read+write` | Read and write access to a token |

:::info
The `values` property is an array of strings. You can use `*` to match all packages or provide exact package names (including scopes, e.g., `@my-scope/my-pkg`).
:::

## Observing All Signals (List Tokens)

You can retrieve all active tokens by sending a `GET` request to the `/-/npm/v1/tokens` endpoint.

```bash
curl -X GET https://my-babadeluxe-registry.dev/-/npm/v1/tokens
```

## Retrieving a Specific Signal (Get Token)

You can retrieve a token by sending a `GET` request to the `/-/npm/v1/tokens/token/:token` endpoint.

```bash
curl -X GET https://my-babadeluxe-registry.dev/-/npm/v1/tokens/token/my-token
```

## Pruning a Signal (Delete Token)

You can delete a token by sending a `DELETE` request to the `/-/npm/v1/tokens/token/:token` endpoint.

```bash
curl -X DELETE https://my-babadeluxe-registry.dev/-/npm/v1/tokens/token/my-token
```
