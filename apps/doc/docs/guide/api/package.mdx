# Package Operations

The API provides the substrate for your shared libraries to be manifest and retrieved. In order to interact with these endpoints, you will need to establish an authenticated connection with the appropriate scope.

## Retrieving a Package (Ingestion)

:::info
In order to retrieve a package, you will need to be authenticated and having at least the `package:read` permission on the targeted package.
:::

You can retrieve a package's metadata by sending a `GET` request to the `/:package` endpoint.

```bash
curl -X GET https://my-babadeluxe-registry.dev/package
# Or for a scoped package:
curl -X GET https://my-babadeluxe-registry.dev/@scope/package
```

:::info
The `:package` parameter is the package you want to get. It can be a simple name or a scoped name.
:::

## Manifesting a Package (Publication)

:::info
In order to publish a package, you will need to be authenticated and having at least the `package:write` permission on the targeted package.
:::

You can publish a package by sending a `PUT` request to the `/:package`. It is usually done by your package manager.

```bash
# Standard package:
PUT https://my-babadeluxe-registry.dev/package
# Scoped package:
PUT https://my-babadeluxe-registry.dev/@scope/package
```

The flow of this publication event involves several synergistic interactions between the Worker brain, D1 for metadata, and R2 for binary storage.
