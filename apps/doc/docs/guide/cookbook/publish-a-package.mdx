# Publishing: A Journey from Local Creation to Registry Nexus

In this brief exploration, we will observe the path of a package from its local inception to its distribution through your Babadeluxe Registry. This process facilitates the seamless sharing of private packages within your team or ecosystem.

## 1. Creating Your Creation
Let us begin by initializing a modest package:

<PackageManagerTabs command="init" />

Next, we shall imbue our package with functionality:

```js title="index.js"
export const add = (a, b) => a + b;
export const sub = (a, b) => a - b;
export const mul = (a, b) => a * b;
export const div = (a, b) => a / b;
```

Now, we must update the `package.json` to acknowledge the registry's presence:

```json {6-9} title="package.json"
{
  "name": "@acme/math",
  "version": "1.0.0",
  "publishConfig": {
    "access": "public",
    "registry": "http://your-babadeluxe-registry.dev"
  },
  "exports": {
    ".": "./index.js"
  }
}
```
Replacing `http://your-babadeluxe-registry.dev` with your instance's unique URI.

## 2. Generating Your Credentials
:::tip
If you have already established a token with the necessary permissions, you may bypass this step.
:::

Consult the [Token Management Reference](/guide/api/tokens) to generate a new credential.

## 3. Establishing Local Authentication
To authenticate with your registry, you must establish an `.npmrc` file with your token:

```txt title=".npmrc"
//your-babadeluxe-registry.dev/:_authToken=YOUR_AUTH_TOKEN
```

## 4. Broadcasting Your Creation 🚀

<PackageManagerTabs command="publish" />

Your package is now part of the Babadeluxe Registry ecosystem, ready to be consumed and evolved.

## 5. Integrating Your Creation

First, ensure your project's `.npmrc` is configured with a token possessing the necessary accesses. Then, install your package:

<PackageManagerTabs command="install --registry http://your-babadeluxe-registry.dev" />

--- 

### Scoped Package Resolution
If you are utilizing scoped packages, such as `@acme/math`, you can configure your `.npmrc` to automatically resolve them via your private registry:

```txt title=".npmrc"
@acme:registry=http://your-babadeluxe-registry.dev
//your-babadeluxe-registry.dev/:_authToken=${BABADELUXE_REGISTRY_AUTH_TOKEN}
```
Storing your `BABADELUXE_REGISTRY_AUTH_TOKEN` within your environment variables for security and portability.

## 6. Evolving Your Creation

When your package undergoes a state change, simply update the version number in `package.json` and publish again to distribute the latest evolution.
