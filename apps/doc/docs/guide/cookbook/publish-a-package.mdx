# Manifesting an Artifact (Publishing)

In this cognitive walkthrough, we will observe how to manifest a package into your Npflared registry node. This process allows for the seamless sharing of private artifacts within your team's collective intelligence 🤝.

## 1. Initializing the Artifact
Begin by creating the substrate for your package.

<PackageManagerTabs command="init" />

Add your cognitive logic to the `index.js` file:

```js title="index.js"
export const add = (a, b) => a + b;
export const sub = (a, b) => a - b;
export const mul = (a, b) => a * b;
export const div = (a, b) => a / b;
```

## 2. Configuring the Connection
Update your `package.json` to signal where the artifact should be manifest.

```json {6-9} title="package.json"
{
  "name": "@acme/math",
  "version": "1.0.0",
  "description": "A mathematical utility for the mesh",
  "main": "index.js",
  "publishConfig": {
    "access": "public",
    "registry": "https://your-npflared-node.dev"
  },
  "exports": {
    ".": "./index.js"
  }
}
```

Replace `https://your-npflared-node.dev` with the actual URL of your registry node.

## 3. Establishing the Trust Signal (Token)
If you haven't yet manifested an authentication token, you can do so via the [CLI](/guide/install/cli) or the [Token API](/guide/api/tokens).

## 4. Authenticating the Node
To establish a persistent connection with your registry, add your token to a `.npmrc` file:

```txt title=".npmrc"
//your-npflared-node.dev/:_authToken=YOUR_SYNERGETIC_TOKEN
```

## 5. Initiating the Publication 🚀
Execute the publication command to manifest your package in the registry.

<PackageManagerTabs command="publish" />

Your artifact is now part of the registry's collective memory, ready to be ingested, upgraded, and consumed by other nodes 🔥.

## 6. Assimilating the Artifact
To ingest the package into another project, ensure your `.npmrc` is configured correctly, then run:

<PackageManagerTabs command="install --registry https://your-npflared-node.dev" />

### Scoped Resolution
For scoped packages (e.g., `@acme/math`), you can configure your `.npmrc` to automatically resolve the scope through your private node:

```txt title=".npmrc"
@acme:registry=https://your-npflared-node.dev
//your-npflared-node.dev/:_authToken=${NPFLARED_TOKEN}
```
